# Database Setup Guide

## Option 1: Automatic Setup (Recommended)
The application will automatically create the `users` table when you run it for the first time. Just start your Go application:

```bash
go run main.go
```

The application will:
1. Connect to Oracle database
2. Check if the `users` table exists
3. Create the table if it doesn't exist
4. Create indexes for better performance
5. Insert sample data

## Option 2: Manual Setup
If you prefer to create the table manually, connect to your Oracle database and run the following SQL:

### Connect to Oracle Database
```bash
# Using SQL*Plus
sqlplus scott/tiger@localhost:1521/FREEPDB1

# Or using SQL Developer, DBeaver, etc.
```

### Run the SQL Script
```sql
-- Create users table
CREATE TABLE users (
    ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    NAME VARCHAR2(100) NOT NULL,
    EMAIL VARCHAR2(255) NOT NULL UNIQUE,
    PHONE VARCHAR2(20),
    STATUS VARCHAR2(20) DEFAULT 'active' CHECK (STATUS IN ('active', 'inactive', 'suspended')),
    CREATED_AT TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Create indexes
CREATE INDEX idx_users_email ON users(EMAIL);
CREATE INDEX idx_users_status ON users(STATUS);

-- Insert sample data
INSERT INTO users (NAME, EMAIL, PHONE, STATUS) VALUES 
('John Doe', 'john.doe@example.com', '+1234567890', 'active');

INSERT INTO users (NAME, EMAIL, PHONE, STATUS) VALUES 
('Jane Smith', 'jane.smith@example.com', '+0987654321', 'active');

INSERT INTO users (NAME, EMAIL, PHONE, STATUS) VALUES 
('Bob Johnson', 'bob.johnson@example.com', '+1122334455', 'active');

-- Commit changes
COMMIT;

-- Verify table creation
SELECT COUNT(*) as user_count FROM users;
SELECT * FROM users ORDER BY CREATED_AT DESC;
```

## Troubleshooting

### Common Issues:

1. **ORA-00942: table or view does not exist**
   - The table hasn't been created yet
   - Run the application (it will auto-create) or create manually

2. **ORA-01031: insufficient privileges**
   - Your user doesn't have CREATE TABLE privileges
   - Connect as a user with proper privileges or ask your DBA

3. **ORA-00955: name is already used by an existing object**
   - Table already exists
   - Check if it was created successfully: `SELECT * FROM users;`

4. **Connection issues**
   - Verify your `.env` file has correct database credentials
   - Test connection: `sqlplus scott/tiger@localhost:1521/FREEPDB1`

### Verify Setup
After setup, test your API:

```bash
# Test health endpoint
curl http://localhost:8080/api/v1/health

# Test get all users
curl http://localhost:8080/api/v1/users
```

You should see the sample users in the response.

## Database Schema

The `users` table structure:

| Column | Type | Constraints |
|--------|------|-------------|
| ID | NUMBER | PRIMARY KEY, AUTO-INCREMENT |
| NAME | VARCHAR2(100) | NOT NULL |
| EMAIL | VARCHAR2(255) | NOT NULL, UNIQUE |
| PHONE | VARCHAR2(20) | Optional |
| STATUS | VARCHAR2(20) | DEFAULT 'active', CHECK constraint |
| CREATED_AT | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP |
